Custom Trades

The custom trades section allows you to make trade algorithms besides just the usual.

There are 4 parts to each array, and you can have as many arrays as you'd like to get the desired trade values

  1. Start: The Start is the starting value for the array. This number is required, and if missing, the whole array will be ignored. This value must be positive. 0 (zero) is allowed in the first entry only.
  2. End: The End is the ending value for the array. If this value is missing, the Start value of the next array will be used as this array's ending value. (Or in the case of an infinitely recursing value, will go on forever). This value must be positive.
  3. Step: The Step value is what increments to use to get from Start to End. If this value is missing, it will make one large jump from Start to End. This value can be negative. In the case of a negative Step with no End point, the value will decrease to 0 where it will stop.
  4. Times: This value is like a repeater. If End and Step are empty, and this value is set, the Start value will repeat the number of Times shown. For instance, if this value is 3, the Start value will repeat 3 times.

A few things to be aware of:

  • If any values are negative (except for Step), they will be ignored
  • End and/or Step will invalidate Times, if either are set, the Times value will have no effect
  • If End is in the wrong direction from Step (for instance, Step = -1, but End is larger than Start), Step will be ignored.

Here are a few examples:

The normal trade values are 4, 6, 8, 10, 12, 15, 20, 25, 30, etc. and the arrays to create this are as follows:

StartEndStepTimes
41220
15050

Note the missing End value for the last array, this means the value will increase by 5 every trade for the rest of the game.

Also note that the End value for the first array does not necessarily have to match the Start value for the second array. If the End value for one array is given as well as a Start value for the next array, both vlaues will be used, even if they are the same.

An example of repeating values:

StartEndStepTimes
4005
10055
500-100

The preceding array will give the following trade values: 4, 4, 4, 4, 4, 10, 15, 20, ..., 40, 45, 50, 40, 30, 20, 10, 0, 0, 0, ...

If you'll note in the second array, the Times value was ignored, and in the third array, no End value was given, so the trade value dropped to 0 and stayed.

An example showing start and end value behavior

StartEndStepTimes
41020
1050100
5010000

This array set will give the following trade values: 4, 6, 8, 10, 10, 20, 30, 40, 50, 50, 100, 100, 100, ...

Notice the duplicated 10 and 50 value. This is because there is an End value and Start value next to each other. If you only want one, you can omit the End value, and it will only use the 10 and 50 once. Example:

StartEndStepTimes
4020
100100
5010000

This array set will give the following trade values: 4, 6, 8, 10, 20, 30, 40, 50, 100, 100, 100, ...

Note that 10 and 50 are used only once in the second values shown.

Also note the single large jump from 50 to 100. As well as the fact that the value stays at 100 for the rest of the game.

If you set the first Start value to 0 (zero), the trade values will start at 0. This is the only place you can set a 0 Start value. You can also leave all other entries blank with a 0 Start value, and this will turn off trading as follows:

StartEndStepTimes
0
